import java.util.Map;

/**
 * 1,统一结果返回
 * 2,使用return this的方式实现了链式调用
 */
public class Result {
    //返回状态码统一
    public static final Integer SUCCESS=20000;
    public static final Integer ERROR=20001;
    //禁止瞎使用对像返回数据,只允许通过调用success或者error方法的任何一个来创建对象
    private Result(){

    }
    /*
    1,是否成功
    2，返回描述
    3，返回码
    4,返回的map
     */
    private Boolean isSuccess;
    private String message;
    private Integer code;
    private Map<String,Object> data = new HashMap<>();
    //成功方法
    public static Result success(){
        Result result = new Result();
        result.setSuccess(true);
        result.setCode(Result.SUCCESS);
        result.setMessage("成功");
        return result;
    }

    public Map<String, Object> getData() {
        return data;
    }



    //失败方法
    public static Result error(){
        Result result = new Result();
        result.setSuccess(false);
        result.setCode(Result.ERROR);
        result.setMessage("失败");
        return result;
    }
    //返回数据
    public Result data(String description,Object result){
        this.data.put(description,result);
        return this;
    }

    //设置不同的状态码
    public Result code(Integer code){
        this.setCode(code);
        return this;
    }




    public Boolean getSuccess() {

        return isSuccess;
    }

    public void setSuccess(Boolean success) {
        isSuccess = success;
    }

    public String getMessage() {
        return message;
    }

    public void setMessage(String message) {
        this.message = message;
    }

    public Integer getCode() {
        return code;
    }

    public void setCode(Integer code) {
        this.code = code;
    }
}

注意：

  • 所谓逻辑删除，它和物理删除相对应，物理删除是直接从表里删除数据，而逻辑删除的本质并不是删除，我们通过设置一个deleted字段，当它的值为1是表示它已被删除，为0是表示它未被删除(默认)，而且，为1时无法被查询到，但是数据库中的数据还在。因此，逻辑删除是一次update操作，只是把deleted的值更新了。
  • 我们当然可以自己制定标志数，@TableLogic(value = “0”,delval = “1”)0表示未删除，或者是在配置文件中修改

JDBC概述

( Java DataBase Connectivity )

就是java连接数据库·····

  • 数据库的种类很多，不可能每换一个数据库都创建一套不同的java代码，因此，SUN公司开发了一套叫JDBC的东西，事实上 ，JDBC就是一套接口，然后让数据库厂家去写实现类，这样java程序员直接调用他们写的实现类就可以了，而这些实现类有个洋气的名字，叫做数据库驱动。

  • 既然是一套接口，那我们知道怎样使用就行，也就是著名的面向API编程😂。
