遇到的问题

1，如何指定为运行时？为什么要指定为运行时？指定其他的有什么作用？

  • @Retention(RetentionPolicy.RUNTIME)可以指定为运行时。
  • 总共有三种环境：SOURCE（源码环境）、CLASS（字节码环境）、Runtime（运行时环境），暂时可以粗略理解成，反射是在运行时环境拿注解，必须保证运行时注解还存在。
  • @Retention决定注解能活到哪个阶段；指定为其他环境的作用待深入理解，涉及字节码工具和编译器。

2，注解的属性为什么是方法的形式？为什么不能写成private的方法？

  • 注解的本质是一个接口，写一个注解相当于写了一个接口继承了java.lang.annotation.Annotation接口，因此：
    • 接口的方法都是默认public Abstract的，所以不能用private修饰

软链接，硬连接

1，ln -s 源文件 目标文件，可以建立软链接，目标文件相当于一个源文件的快捷方式

  • 软链接的文件相当于一个路径，指向文件名

  • 源文件最好使用绝对路径，不然目标文件变动后，找不到源文件

  • 修改目标文件的内容会同步到源文件

  • 源文件移动后，软链接会断开

2，linux文件系统，文件名和数据完全分开

  • 路径层，通过路径解析可以找到目录层的指定目录
  • 目录层，目录层存放文件名和iNode的映射
  • inode层，通过iNode中的元数据找到真实数据存的地方
  • 软链接相当于一个路径，指向一个目录，当目录中文件名变了，软链接就断了
  • 删除原文件也只是删除了源文件的目录层，iNode和数据都还在，所以对硬链接没影响，由于没有目录，路径就解析不到目录，所以软链接就断开

2，ln 源文件 目标文件，建立硬连接

  • 源文件移动或者

  • 硬连接相当于在目录新增一个文件名，和原来的文件名映射同一个iNode，就算原来的文件名删除，文件数据也依旧可以通过这个文件名访问

  • 当没有文件名映射到这个iNode的时候，iNode元数据指向的真实数据也会被删除

软件安装

1，安装，sudo apt install 软件名

2，删除，sudo apt remove 软件名

3，设置源，源就是保存了apt所有安装包的仓库

  • 设置为阿里云镜像 sudo sed -i ‘s/archive.ubuntu.com/mirrors.aliyun.com/g’ /etc/apt/sources.list
    • sed表示直接对文件做流式操作 s代表替换，把文件中archive.ubuntu.com替换为mirrors.aliyun.com，g代表全局
  • 设置 安全镜像阿里云 sudo sed -i ‘s/security.ubuntu.com/mirrors.aliyun.com/g’ /etc/apt/sources.list
    • 安全镜像负责安装后软件的安全和漏洞更新
  • 重新设置apt源的索引 sudo apt update，更新apt索引中软件的信息
    • sed：文本编辑工具
    • -i：直接修改原文件
    • 's/archive.ubuntu.com/mirrors.aliyun.com/g'：替换规则
    • /etc/apt/sources.list：要修改的文件
  • 软件是用户共用的，需要root权限才能操作

4，更新软件 sudo apt upgrade，真正更新软件版本

2，显示宽度和存储宽度

  • 存储宽度：指的是数据类型可以存储的最大范围，比如int是4字节，tinyint是1字节，是固定的
  • 显示宽度：指的是指定显示成几位，不够的补0，比如定义数据类型int(4)，显示4位，0001,0055等
    • 注意：tinyint(1)在不同的客户端可能有不同的显示，在老版本中tinyint(1)可能会被读取成布尔类型，如果服务器使用的是数字代表布尔，那么不要用tinyint(1)，直接使用tinyint；在msyql8后用Boolean类型替代了它。
