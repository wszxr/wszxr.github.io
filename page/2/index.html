8.逻辑删除

  • User类中deleted属性对应字段deleted
  • 在配置类中配置插件
  • 这样做之后所有的删除都会变成逻辑删除

注意：

  • 所谓逻辑删除，它和物理删除相对应，物理删除是直接从表里删除数据，而逻辑删除的本质并不是删除，我们通过设置一个deleted字段，当它的值为1是表示它已被删除，为0是表示它未被删除(默认)，而且，为1时无法被查询到，但是数据库中的数据还在。因此，逻辑删除是一次update操作，只是把deleted的值更新了。
  • 我们当然可以自己制定标志数，@TableLogic(value = “0”,delval = “1”)0表示未删除，或者是在配置文件中修改

12.乐观锁

1.悲观锁和乐观锁

  • 悲观锁就类似于串行运行，在我事务提交前，你只能等着。
  • 乐观锁给操作对象加一个等级，当有两个人一起操作同一个对象时，首先他们都会获得该对象的等级，如果谁先提交了事务，那么对象的等级就会加一，他们在事务提交的时候会验证对象的等级是否和前面一样，如果不一样就提交事务失败，就会回滚。

2.实现

  1. 在表中加一个字段标识等级

  • 用@Version标识属性，并且在插入新数据的时候指定version为1

JDBC概述

( Java DataBase Connectivity )

就是java连接数据库·····

  • 数据库的种类很多，不可能每换一个数据库都创建一套不同的java代码，因此，SUN公司开发了一套叫JDBC的东西，事实上 ，JDBC就是一套接口，然后让数据库厂家去写实现类，这样java程序员直接调用他们写的实现类就可以了，而这些实现类有个洋气的名字，叫做数据库驱动。

  • 既然是一套接口，那我们知道怎样使用就行，也就是著名的面向API编程😂。

实现步骤

我们在java中进行操作大致需要

1.导包

2.注册驱动

3.获取连接

4.定制要执行的sql语句

5.获取执行语句的对象

6.执行

7.处理结果

8.释放资源

  • 看着步骤多，实则很简单

代码实现

  • 这是一套基本流程，既然是面向API编程，那我们把注意力移到API上
