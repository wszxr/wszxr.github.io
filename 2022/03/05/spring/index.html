后置处理bean

红色部分

在Bean初始化的前后可以实现再加工。

实现步骤

  1. 用自己的类实现BeanPostProcessor接口，然后在方法中写入自己的加工代码。

  1. 在配置文件制造自己写的类

    • 需要注意的是，实现了这个类后，工厂中所有创建的对象都会调用这个方法，会出现类型不匹配问题，因此要筛选出自己需要加工的类
    • 一般只在postProcessAfterInitialization方法中加工，但是别忘了postProcessBefInitialization也会执行，所以必须让它return传入的对象。

静态代理模式

1.代理设计模式：通过代理给原始类增加额外的功能，有利于原始类的维护

静态代理编码

代理类 = 目标类(原始类) + 额外功能 +和原始类(目标类)实现相同的接口

1.这是目标类，实现了UserService接口

2.这是接口中的方法，由Person类实现

3.用代理增加额外的功能，这些功能可能会经常更改，所以不写在目标类

  • ```java
    //为了保持和目标类的方法一致，代理类和目标类实现同一个接口
    public class PersonProxy implements UserService{
    //需要获取一个目标类，这里new了一个，因为每个Person的接口方法是一样的，具体使用时可以传一个实例进来
    Person person = new Person();
    @Override
    public void service() {
    person.service();
    //除了调用增加功能之前的方法，额外加上新的功能
    System.out.println(“代理service”);
    }
    }

静态代理类的缺点

  1. 每一个实现类都需要手动创建一个代理类，类太多不利于项目管理
  2. 代理类中更改代码过于麻烦

  • 通过代理为原始类增加新的功能
  • 利于原始类的维护
