工厂设计模式

  1. 概念：创建对象交给工厂，而不是自己new

  2. 优势：解耦合

耦合

  • 指代码间的强关联关系，一方的改变会影响另一方。
  • 耦合带来的问题：不利于代码的维护。

通用工厂的设计

  • 首先，不用工厂模式创建对象

  • 用工厂模式创建

    • 这里直接实现通用的工厂

    • 在resoures根目录下创建Application.properties文件

    • person1 = main.demo1.Person

    • @Test方法

    • @Test
    public void test1(){
        //用工厂类的静态getBean方法获取实例并输出地址
        Person person = (Person) BeanFactory.getBean("person1");
        System.out.println(person);
    }
