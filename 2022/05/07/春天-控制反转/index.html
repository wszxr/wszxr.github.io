springFramework(春天)

1,为什么要有spring，什么是spring

EJB(Enterprise Java Bean)的问题

1，运行环境苛刻，无法在tomcat中运行，需要在EJB容器中才能运行，并且是收费的。

2，代码移植性差，实现了固定容器的接口，所以没法在其他环境运行。

3，综上所述，EJB是重量级的框架。

什么是spring

是javaEE轻量级的一个解决方案

1，为什么是轻量级

  • 运行环境没要求。

  • 代码移植性高，因为不需要实现额外的接口。

2，什么叫解决方案

可以解决javaEE分层开发的每一层的问题，而不是像mybatis等只负责一层。

整合了设计模式

1，绝对的核心，工厂模式

实例工厂

为什么有了FactoryBean还要有实例工厂？

1，FactoryBean是spring提供的接口，使用的话会导致spring侵入自己的应用

2，主要是遗留系统

  • connFactory是遗留的工厂类，类中getConnection(）方法可以获得对象，没有给我们代码，所以无法继承FactoryBean接口，我们只需要先创建工厂bean，再在bean标签指定工厂bean和方法就好了。

静态工厂

如果工厂类的工厂方法是静态的，那么就可以省略掉创建工厂对象的步骤，直接指定工厂类和方法即可。

3，工厂总结

  • 如果不是遗留系统，还是建议使用FactoryBean

7，对象创建次数

  • 简单对象以<bean 标签中的 scope属性控制 singleton/prototype对应单例/多例，默认singleton
  • 复杂对象用isSingleton()方法控制
  • 实例工厂和静态工厂也采用scope属性控制

8，bean生命周期

创建

1，如果是单例对象，工厂被创建(new出来)是对象也被创建了。在bean标签中设置懒初始化(lazy-init=true)可以让singleton对象在获取对象时再被创建

2，如果是多例，获取对象的时候(ctx.getBean())，对象才会被创建

初始化

对象初始化时spring会调用初始化方法

初始化方法的书写：

1，实现InitializingBean接口

然后配置文件中交给spring管理

2，随意指定某类中某方法

交给spring管理并指定为初始化方法

细节：

  • 如果以上两种都实现了，先执行接口的方法，再执行自己指定的
  • 初始化发生在属性注入之后
  • 一般初始化方法里放资源的初始化，数据库，IO等，但是一般不用。

销毁

1，实现DisposableBean接口，然后把对象交给spring管理

2，直接指定方法，通过destroy-method属性指定方法

3，ctx.close()方法调用时执行销毁操作

  • 只适用于scope=”singleton”的对象
  • 销毁主要指的是资源的释放
  • 和初始化一样基本不用

10，自定义类型转换器

类型转换器

配置文件中的value都是字符串，但是却可以注入到类中的其他类型，这是spring内置的一些类型转换器的功劳。

自定义类型转换器

1，我们在Person中加一个Date的birthday字段，然后再xml中赋值。

会出错，转换不了

  • 所以说，当spring没有提供相关转换器时，就需要自己开发自定义类型转换器了。

2，自定义步骤

  • 自定义类实现Converter<>接口，泛型填的是转换前和转换后的类，把自定义转换的代码写到convert方法内。
  • 在配置文件中把这个类交给spring管理
  • 要想让spring知道你这是个类型转换器，就必须注册

ConversionServiceFactoryBean类：看图很容易知道，converters是一个Set集合，且有set方法，所以可以通过property注入(注册)

  • 这几步之后，只要遇到需要注入的是Date类，就会使用这个转换器

结果：转换成功了

细节：

  • 事实上，java内置了Date转换器，但要求格式必须是”yyyy/MM/dd”

  • ConversionServiceFactoryBean类的id必须是conversionService
