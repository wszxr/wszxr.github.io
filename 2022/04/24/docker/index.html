docker好处

1.环境一致

2.进程隔离，操作系统层面

3.镜像机制

安装docker

  1. 通用方式安装，所有linux平台都可以
  2. 下载脚本curl -fsSL get.docker.com -o get-docker.sh
  3. 设置阿里云镜像并下载 sh get-docker.sh –mirror Aliyun
  4. 启动
  • systemctl enable docker 开机自启动
  • systemctl start docker 启动
  • restart 重启 status 状态 stop 停止

5，入组

  • 创建docker组 groupadd docker
  • 当前用户加入docker组 usermod -aG docker $USER

6,查看

  • docker version 版本
  • docker info 详细信息
  1. 镜像，一个镜像就是一个软件，例如：mysql，redis
  2. 容器，镜像每一次运行都会产生一个容器，是一个运行的软件服务
  3. 镜像获取，docker仓库，远程 docker hub
  4. 本地仓库，类似maven的本地仓库，默认/var/lib/docker

  • 把远程仓库换成阿里云的镜像
  1. 阿里云镜像服务
  2. 创建/etc/docker文件
  3. 粘贴进去

镜像命令

  1. docker image ls ==== docker images ，docker images -q列出id

  3. 定位镜像 名称:tag 或者镜像id

  4. 下载镜像 docker pull 名称:tag,不指定tag默认最新版

  5. 看当前镜像是否存在 docker search 名称

  6. 删除

    • 正常删除 docker image rm 名称:tag 或id，要求必须是没有运行过的镜像
    • 强制 docker image rm -f ··········

  7. 镜像备份恢复

    • docker save 镜像名：tag -o 路径/xxxx(推荐name-tag).tar
    • docker load -i（image）xxxx.tar

容器相关

格式: docker 命令 [选项]

1,

  • docker ps查看运行中的容器

  • 查看所有 docker ps -a
  • docker ps -a -q查看所有容器id

2,

  • docker run 名称:tag||id 需要把容器端口和宿主机端口（linux）端口映射才能访问
  • docker run -p 8080（宿主机）:8080(容器内)名称:tag ，需要开启防火墙，且可以映射多个端口
  • 后台启动（守护进程）-d—–》 docker run -p 8080（宿主机）:8080(容器内) -d 名称:tag
  • 起名字 –（两个-）name 名称（唯一）

3，停止 重启 暂停 恢复

  • docker stop id||名称：tag
  • ···restart···
  • ···pause···
  • ···unpause···

4，杀死容器 docker kill id||名称：tag

5，删除 容器

  • docker rm id···（正在运行的不能删除）
  • docker rm -f ···（强制删除）
  • docker rm -f$（docker ps -q -a）

6,查看日志

  • docker log -f(实时) -t(时间) id||name

7，进去容器内部

  • docker exec -it（交互模式） id||name bash(进入bash窗口)
  • exit退出容器

8，宿主机和容器之间文件拷贝

  • docker cp 容器id:文件或目录 宿主机目录
  • docker cp 宿主机文件或目录 容器id:文件或目录

9，查看容器内进程

  • docker top id||name:tag

10,查看容器详细细节

  • docker inspect name

11,数据卷机制（Data volume）

  • 作用：实现容器中的数据和宿主机映射（同步）
  • 把宿主机中的某目录和容器中的目录相绑定
  • 要在容器首次启动时就使用
  • docker -v 宿主机目录绝对路径：容器目录：ro
    • 如果宿主机目录是绝对路径，那么会删掉容器中目录的所有原始内容
    • ：ro(readonly) 容器中目录只读
  • 使用别名替换宿主机绝对路径可以保留容器目录中的内容
    • docker -v xxxx:容器目录
    • docker根据别名创建的目录/var/lib/docker/volume/xxxx

12,将容器打包成一个新的镜像，深度定制自己的容器。运行时直接存在定制的特性

  • docker commit -m “描述信息” 容器id||name -a “作者” id 打包后name:打包后版本

常见软件服务

mysql

  1. pull拉取
  2. 运行 docker run -p3306:3306 -e MYSQL_ROOT_PASSWORD -d –name=mysql01 –restart=always -v /root/data:/var/lib/mysql mysql:5.6
  3. -p端口映射|| -e环境设置|| -d后台运行|| –name起名字|| –restart=always docker重启容器也重启||-v 数据卷-v /root/data:/var/lib/mysql=》宿主机的 /root/data绑定容器的/var/lib/mysql
  4. 持久化通过数据卷完成
  5. docker中的mysql自动开启远程连接
  6. 数据卷将底层的数据库文件系统进行了备份，不利于数据迁移，应该导出为sql文件
  7. $ docker exec some-mysql sh -c ‘exec mysqldump –all-databases -uroot -p”$MYSQL_ROOT_PASSWORD”‘ > /some/path/on/your/host/all-databases.sql
    • –all-databases表示所有数据库的数据
    • /some/path/on/your/host/all-databases.sql表示输出路径
    • -c 是不交互，some-mysql是某个mysql容器，可以写id
    • -uroot -p”$MYSQL_ROOT_PASSWORD”表示自动使用设置的管理员密码登录

redis

  1. pull拉取
  2. 运行 docker exec -it 40d bash
    • docker run –name redis01 -d -p 6379:6379 –restart=always -v /root/redisdata:/data redis:5.0.12 redis-server –appendonly yes
    • redis-server –appendonly yes是开启aof持久化
    • -v /root/redisdata:/data是把默认持久化的文件目录绑定到宿主机的/root/redisdata里面
  3. 连接 redis-cli
  4. 操作

5，自定义配置文件启动

  1. ​ 传文件方式,会覆盖docker所有的默认配置，不推荐
    • 下载redis压缩包获取redis.config文件
    • 传到宿主机的某目录中
    • 通过数据卷绑定传输给容器内
    • run的时候把redis.config文件的路径放到redis-server后面
    • 注意需要开启远程调用，把config中的bind属性改成0.0.0.0
  2. 推荐部分覆盖，创建redis.config文件，把要修改的部分写上就行，其他步骤和上边一样

nginx

  1. pull镜像
  2. docker run -p 80:80 -d –restart=always –name nginx1 -v/宿主目录/nginx.conf:/etc/nginx/nginx.conf
    • 由于不能覆盖目录下的其他文件，所以单文件数据卷绑定
  3. 需要把容器中的nginx.conf复制到宿主机的目录下，然后更改
  4. 作为服务器，数据卷挂载家目录（index.html）所在的目录

容器间通信

使用网络

  1. 用-p（端口之间的映射）完成容器与宿主机之间的通信
  2. 用容器内ip地址完成容器和容器之间的通信，容器ip是动态生成的，不推荐使用这种方法
  3. 使用容器名完成容器之间的通信，注意不能使用默认的docker0网桥
  4. 网络（网桥）创建

高级数据卷（volume)

  1. 用于实现宿主机和容器之间的目录或文件映射
  2. -v 绝对路径||别名：容器目录路径
    • 绝对路径，不会保留容器中原始数据
    • 别名，会保留容器中原始数据
  3. 别名是什么？
    • docker维护的数据卷
  4. docker volume ls列出
  5. docker volume inspect 别名，查看详细信息
  7. docker rm 别名，删除

Dockerfile

  1. 定义：镜像的描述文件
  2. 作用：通过Dockerfile可以构建属于自己的镜像
  4. 指令：docker build -t 镜像名：版本（tag）.(指定Dockerfile文件所在的位置)
  6. 保留指令

docker compose

  1. 一个Docker官方的开源项目，负责对容器的编排
  2. 项目:一组关联的应用容器组成的单元，对应一个docker-compose.yml
  3. 服务:单元中的每一个容器都是一个服务
  4. Linux在安装docker的时候不会默认安装docker-compose，windows和macos会
  5. 安装
    • 离线安装
      • 下载docker-compose脚本，放入/usr/local/bin
      • 修改权限 chmod +x /usr/local/bin/docker-compose
    • 在线安装
      • 从远端仓库下载
      • 修改权限
  6. docker-compose.yml

模板指令

  1. build指令，启动前先通过docker-file构建镜像，
  2. command指令，覆盖cmd的变量
  3. container_name,指定容器名称，不建议使用
  4. depends_on指令，依赖的服务会先启动，不会等到完全启动再启动
  5. environment 环境指令，写死在docker-compose.yml中
  6. env_file指定环境文件
  7. image 指定启动容器使用的镜像
  8. networks 指定启动容器时的网桥，如果不能再则需要声明网桥
  9. ports 指定端口映射 宿主机：容器
  10. volumes 数据卷 容器目录：宿主机目录 使用别名绑定需要声明
  11. restart 容器会随着引擎重启而重启
  12. 使用参考

docker-compose 命令

  • 以下服务id指的是docker-compose.yml中的服务名字
  1. docker compose up 启动所有容器
  2. docker-compose up[id]针对于某一个服务（容器）
  3. ============以下都是docker-compose
  4. down 关闭项目并删除网桥
  5. exec 服务id 可以进入服务（容器）内部
  6. ps（列出运行服务）
  7. restart 服务id
  8. rm（删除停止的容器） -f(强制删除)
  9. top （查看项目所有服务中的进程） 服务id（查看指定的）
  10. stop，pause（暂停），unpause（恢复）
  11. logs（查看整个项目的日志） 服务id（指定某个服务） -f（实时）
