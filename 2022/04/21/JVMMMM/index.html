JVM

1，内存结构

程序计数器（pc Register）

  1. java运行流程：
    • java源码–>jvm指令–>解释器–>机器码–>cpu执行
  2. 计数器作用：记录下一条指令的地址
  3. 特点：
    • java把寄存器当作程序计数器，是唯一不会出现内存溢出的问题的区域
    • 线程私有，每一个线程有自己独立的程序计数器

虚拟机栈（jvm Stacks）

  1. 定义：
    • 每个线程运行所需要的运行空间。
  2. 组成：
    • 由若干栈帧组成，每个栈帧对应一个方法的内存空间（局部变量，参数，返回地址···）。
    • 每个栈只有一个活动栈帧(顶部)，对应正在运行的那个方法。
  3. 栈内存溢出：
    • 单个栈帧内存过大(一般不会出现)。
    • 栈帧过多(递归时没有定义结束条件时)。
  4. 注意：
    • 垃圾回收不涉及栈内存，方法运行完自动释放。
    • 单个栈内存并不是越大越好，越大，能创建的线程数就越小。
  5. 方法中的局部变量是否有线程安全问题：
    • 静态的变量是共享的，线程不安全。
    • 通过传递参数进来的是不安全的。
    • 逃离出方法作用范围的变量不安全，例如return 出去的变量。
    • 方法内非静态基本类型变量是线程安全的，每个线程都有独立的一份。

本地方法栈（Native Method Stacks）

  1. Java中本地方法调用时使用的内存。

堆（Heap）

  • 通过new关键字创建的对象会使用堆内存。
  • 堆是线程共享的。
  • 堆中有垃圾回收机制.
  1. 堆内存溢出：
    • 当对象没有被任何一个引用所引用，jvm才会把它当作垃圾。
    • OutOfMemoryError。

方法区（Method Area)

  1. 定义（jvm1.8）：

    • 是一种规范，存储和类的结构相关的信息，运行时常量池。
    • 在虚拟机启动时被创建.
    • 在逻辑上其实是堆的一部分,可以不遵守此规范（由具体jvm实现决定）。
      • 例如：jvm1.8以下是在永久代里边
      • jvm1.8以上是在本地内存中的元空间里

  2. 溢出：

    • jvm1.8以前是永久代溢出。
    • 1.8以后是元空间溢出，元空间属于操作系统的空间。
    • 溢出场景：
      • spring和mybaties用cglib创建动态代理类。

  3. 常量池：

    • 就是在字节码文件中的一张表。
    • 作用，给指令一个符号，用于虚拟机寻找要执行的东西。

  4. 运行时常量池：

    • 常量池在字节码文件运行时加载到内存中的运行时常量池中，这时候符号变成了真实的内存地址。

    • StringTable（字符串常量池）:

      • 是一个hashtable结构而且不可以扩容。

      • 当第一次定义字符串时，如果串池中没有则会把对象放入串池中，如果有则直接使用串池中的对象。

      • 示例

      • /**
 * 关于串池的题
 */
public static void main(String[] args) {
    String s1 = "a";
    String s2 = "b";
    //常量拼接
    String s3 = "a"+"b";

    //s4,变量拼接实际上是新建一个空的StringBuild(),然后.append("a").append("b").toString
    //toString会把拼接好的值放到一个新的对象中，然后复制给s4
    String s4 = s1+s2;


    String s5 = "ab";

    //尝试将字符串对象放入串池，如果串池里面有相同的常量，就不加入，
    // 如果没有，就加入（jdk1.8以下会拷贝一份再加入拷贝的对象，拷贝出来的对象和原对象不一样），
    //无论加入或者是没加入都会返回串池中字符串常量对应的对象
    String s6 = s4.intern();

    System.out.println(s3==s4);//false =>             false=>//s3的对象在串池中，s4的在堆里面
    System.out.println(s3==s5);//true  =>正确答案      true=>因为String是不可变的，而且都是常量，因此结果是确定的，在编译期间就优化成"ab"
    System.out.println(s3==s6);//true  =>             true
}

      • 题目总结：

        • 当串池中没有时，新建一个String对象会加入串池，之后创建的字符串常量如果是一样的，那么就统一用串池中的那个对象。
        • 字符串常量对象拼接会在编译期间优化成拼接后的常量；而字符串变量拼接使用的是StringBuild，产生的对象存在堆中。
        • intern()方法会尝试将堆中的对象放入常量池，串池中有此常量，那就不加入，没有则加入(java1.8)；无论加入成功或者是失败方法返回值都是串池中对应常量的对象。

      • StringTable的位置：

        • java1.7的StringTable从永久代转到了堆空间中，因为永久代只有fullGC时才会执行回收。

      • StringTable也会被垃圾回收。

      • StringTable调优：

        • StringTable实际上是一个hashTable，根据实际情况，调整桶的个数就是调优了，桶的个数的最小值是1009。

直接内存(DirectMemory)

  1. 定义：

    • 常用于NIO操作，作为数据缓冲区。
    • 分配和回收此内存空间成本较高，但是读写性能高。
    • 不受jvm回收管理，因为是操作系统的内存。

    • 直接内存，java代码可以直接访问，比原始io少了一次缓存复制的过程。

  4. 直接内存的分配和回收都由unsafe对象完成：

    • 以ByteBuffer为例：它的构造器中获取了unsafe实例，并且调用unsafe实例的allocatememory（）和setmemory（）两个方法分配了直接内存。然后还使用Cleaner（虚引用对象）监测ByteBuffer实例，当ByteBuffer被GC回收时，会调用Cleaner的clean(),进而调用clean()中的unsafe.freeMemory()方法对直接内存进行释放。
    • java中默认禁止了显式GC，因为System.gc()是fullGC，会是程序停留时间过长。这时候只能等待jvm进行GC，或者是直接使用unsafe对象的.freeMemory()方法。
    • Unsafe只能通过反射强制获取实例。
