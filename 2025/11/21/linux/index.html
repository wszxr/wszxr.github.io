linux内核

Linux目录

1，/表示根目录，Linux只有一个盘

  • bin：存放可执行文件
  • etc：存放配置文件
  • home：每个用户的家文件夹

2，操作应该在home的用户文件夹里

文件相关命令

ls

1，作用：列出目录下的文件和目录

2，使用

  • ls -a：包含隐藏文件，隐藏文件是.开头的文件
  • ls -l：列出文件的详细信息
  • ls -h：配合 -l使用，让大文件用k或m显示，而不是只用字节显示
  • ls -lha:可以合在一起

3，通配符

  • ？代表任意一个字符
  • *代表任意多个字符，可以是0个
  • [1,2,3,5] 代表一个字符，只能是指定这几个中的一个
  • [1-4] 等同于[1,2,3,4]，代表一个字符

cd change direction

1，作用：切换目录

2，使用

  • cd/cd ~ 代表回到用户的家目录，~直接代表家目录，cd ~/zxr
  • cd xxx代表去xxx目录
  • cd .代表当前目录
  • cd ..代表上级目录
  • cd -代表在最近两个目录直接来回切换

创建和删除

1，touch ~/zxr.txt代表在用户的家目录创建一个zxr.txt文件，如果该文件存在，那么会该表该文件的修改时间

2，mkdir ~/zxr代表在家目录创建一个zxr的目录，目录存在，或者目录与文件同名则创建失败

  • mkdir -p ~/zxr/friends表示在家目录创建zxr，在zxr目录再创建friends目录

3，rm a.txt代表删除文件，不能删除目录

  • rm -f代表强制删除，不会提示错误信息
  • rm -r a代表递归删除a目录以及目录以下的文件
  • rm可以使用通配符：rm -rf *，代表删除目录下所有的文件和目录

文件拷贝和移动

1，cp 源文件(路径加文件名) 目标文件(路径加文件名)，代表把源文件复制为目标目录下的新文件

  • cp -i ，在目标目录存在与目标文件名同名的文件时，提示是否要覆盖，y or n
  • cp -r，复制当前目录为新的目录，包含目录底下的目录和文件

2，mv 源文件 目标文件，移动文件并重命名为目标文件

  • mv 源文件 目标文件，如果目标文件就是源文件，则可以重命名源文件为目标文件

3，tree xx，显示xx目录下面的目录文件树

  • tree -d xx，只显示目录

文件查看

1，cat xx.txt代表查看文件内容

  • cat -b 代表显示行号，空行没有行号
  • cat -m代表显示行号，空行也有行号

2，more xx.txt 代表分页查看文件内容

  • 按f或空格查看下一页
  • 按b查看上一页
  • 回车查看下一行
  • q退出

3，less xx.txt是more的增强版

  • 空格键：向下翻一页
  • b 键：向上翻一页
  • 上/下箭头键：向上/下滚动一行
  • g 键：跳到文件开头
  • G 键：跳到文件末尾
  • q 键：退出 less
  • /zxr搜索zxr字符并高亮
  • esc + U，取消高亮

4，grep zxr zxr.txt表示在zxr.txt搜索zxr，显示包含zxr的行

  • grep -n zxr zxr.txt显示包含行号
  • grep -v zxr zxr.txt显示不包含zxr的行号
  • -i表示忽略大小写
  • 使用通配符
    • grep -n ^hf zxrs.txt 查询hf开头的行并显示行号
    • grep -n fh$ zxrs.txt 查询fh结尾的行并且显示行号

echo和重定向

1，echo daniuma执行后会在终端显示daniuma

2，echo daniuma > a.txt 代表把终端展示的内容输出到a.txt，覆盖原有内容

2，>> 追加内容，而不是覆盖

管道

1，把左边命令的输出作为右边命令的参数，相当于在管子左边放东西，从管子右边取出来作为参数

2，ls -lh | grep zxr代表把ls的输出作为grep的参数，grep会筛选出包含zxr的行

ssh远程连接

用户，权限，组

权限

1，linux可以有多个用户，每个用户在home目录都有一个文件夹

2，对于每个文件或目录，都可以对不同的用户设置不同的权限

3，组代表一组用户，每个文件或目录都可以给一组用户设置不同的权限，用于简化权限管理

4，ls -l的解析

  • 1，d代表目录，-代表文件
  • 2，代表文件拥有者的权限，r是读，w是写，x是执行权限；第一个文件拥有者是zxr，第二个是root
  • 3，代表组的权限，默认组名和用户名一样，zxr组对该文件的权限是r-x，可读可执行
  • 4，代表非拥有者和指定组对该文件的权限
  • 5，硬链接，到达此文件或目录的路径数
    • 文件一般都是1，代表用绝对路径访问
    • 目录默认有绝对路径，和在目录中用.访问两种；还可以在下一级用..访问，跨级目录不算因为不存在直接跳2或3级等的指令，因此目录的硬链接数=2+子目录的数量
  • 6，文件拥有者
  • 7，文件所属用户组，一个文件只能属于一个用户组，但是可以属于多个用户
  • 8，文件大小
  • 9，文件最大修改时间

5，设置文件的权限

  • chmod u/g/o/a +/- rwx:
    • chmod代表changemode指令
    • u/g/o/a分别代表给拥有者，拥有组，其他用户，所有用户
    • +/-代表增加，减少权限
    • +r代表增加读权限，+w代表增加写权限，x代表执行权限，可以+wrx等等
  • 可以用数字表示
    • rwx= 4+2+1 = 7
    • rw-= 4+2 = 6
    • r-x= 4+1 = 5
    • r--= 4
    • ---= 0
  • chmod 765 表示u 为7(rwx),g为6(rw-),0为5(r-x)

6，修改目录权限的方法是一样的，不过注意

  • 要cd进目录需要目录的执行权限x
  • 进入目录后要ls显示内容需要目录的刻度权限r

组

1，添加组 sudo groupadd dev，sudo表示用管理员的身份执行

2，删除组 sudo groupdel dev

3，查看组 cat/less /etc/group ，保存的组都会存到etc文件中

用户

1，创建用户，sudo useradd -m -g dev 张三

  • 创建一个名字叫张三的用户，需要root权限
  • -m代表自动创建家目录，且配置家目录的权限
  • -g 指定用户所属的组，如果不指定默认创建一个和用户名一样的组

2，设置密码，sudo passwd 张三 zxr6666

  • 给制定用户设置密码可以使用sudo
  • 修改自己的密码直接passwd就行了

3，删除密码 userdel -r username

  • -r代表同时删除用户家目录

4，查看用户信息 cat /etc/passwd | grep zhangsan

  • 存在root的/ect/passwd文件下

  • 用户信息用：分隔，分别是
    • 用户名：是否有密码(x代表有，默认存在/etc/shadow文件中，加密保存)：用户id：主要组id：用户全名：用户家目录：命令行解释器，负责接收命令并解析执行

5，使用id指令查看用户信息 id zxr

  • uid是用户ID，zxr是用户名
  • gid是主要组的id，指的是useradd -g指定的组，用户创建的新文件都默认属于这个组
  • groups表示用户所属的组集合，用户可以属于多个组，同时拥有多个组的权限

6，使用who和who am i查看用户信息

  • who可以显示当前机器上登录的所有用户
  • pts/0代表远程ssh登录，后面是登录时间和远程的ip地址
  • tty2代表本地登录，直接在机器上登录的

  • who am i 可以返回当前用户信息

7，usermod切换用户的主要组，附加组，使用的shell；需要root权限，改完需要重新连接

  • usermod -g dev zhangsan，把张三的主要组设置成dev

  • usermod -aG sudo，www-data zhangsan，把张三添加进sudo，www-data附加组

    • -G 代表增加附加组
    • -a 代表追加，不写就是覆盖

  • usermod -s /bin/bash zhangsan，把zhangsan登录是用的shell程序切换为bash

    • 推荐使用bash，有文件和目录的颜色区分

8，which用于查看指令程序的存放位置

9，su zxr，用于切换用户

  • su -zxr，切换后到达用户的家目录
  • exit，退出当前用户，如果在root 执行su zxr，再exit会返回到root，都是递归的
  • su 不指定用户和su root 的功能一样，都是切换到root用户

10，修改文件拥有者，文件所属组，文件权限

  • chown zhangsan user.txt，修改文件拥有者为zhangsan
  • chgrp dev user.txt，修改文件组为dev
  • chmod 用来修改权限
  • 以上都可以用于目录或者文件，如果要更改目录里面包含的所有子目录和文件，加上-R参数

系统信息相关命令

1，查看系统时间，date

2，查看系统日历，需要安装ncal软件

  • cal显示一个月的日历
  • cal -y显示一年的日历

3，查看磁盘使用情况 df -h (disk free)

  • -h 用于以能看懂的方式显示字节数
  • 主要关注挂载点的/目录，能直观显示磁盘的使用情况

4，查看目录下文件总大小 du(disk usage) -h

  • 这里有三个目录(./，test，inq)，du -h显示了每个目录下的文件的大小总和

5，查看进程 ps，默认显示当前用户的用终端命令执行的进程

  • ps -a，显示所有用户的进程
  • ps -u，显示进程的详细状态
  • ps -x，显示所有进程，包括非终端执行的
  • ps -aux可以显示所有用户的所有进程

6，动态查看进程top，可以快速筛选出占资源的进程

  • top -o 字段，指定排序字段

  • 会默认按照cpu使用率降序，动态监控进程

  • 按P用cup排序，按M切换内存排序，按T切换时间排序

  • 按q退出top，按R切换为降序

7，杀死进程 kill

  • kill pid，杀死进程，进程会执行完关闭所需要的保存日志等工作
  • kill -9 pid，强制杀死进程，用于进程无响应，不能正常关闭

其他命令

查找文件

1，find path -name “*.txt” ，在path下查找文件名为 .txt的文件， *代表匹配多个字符

  • path不指定默认当前路径
  • 默认递归查找，子目录中有也会被查找到

软链接，硬连接

1，ln -s 源文件 目标文件，可以建立软链接，目标文件相当于一个源文件的快捷方式

  • 软链接的文件相当于一个路径，指向文件名

  • 源文件最好使用绝对路径，不然目标文件变动后，找不到源文件

  • 修改目标文件的内容会同步到源文件

  • 源文件移动后，软链接会断开

2，linux文件系统，文件名和数据完全分开

  • 路径层，通过路径解析可以找到目录层的指定目录
  • 目录层，目录层存放文件名和iNode的映射
  • inode层，通过iNode中的元数据找到真实数据存的地方
  • 软链接相当于一个路径，指向一个目录，当目录中文件名变了，软链接就断了
  • 删除原文件也只是删除了源文件的目录层，iNode和数据都还在，所以对硬链接没影响，由于没有目录，路径就解析不到目录，所以软链接就断开

2，ln 源文件 目标文件，建立硬连接

  • 源文件移动或者

  • 硬连接相当于在目录新增一个文件名，和原来的文件名映射同一个iNode，就算原来的文件名删除，文件数据也依旧可以通过这个文件名访问

  • 当没有文件名映射到这个iNode的时候，iNode元数据指向的真实数据也会被删除

  • 把源文件1，源文件2打包为目标文件

  • c代表打包

  • f指定目标文件，f后必须跟目标文件

  • 2

  • x代表解包

压缩解压

1，gzip -k 01.py ，压缩01.py文件，只能压缩文件，不能压缩目录

  • -k表示保留源文件
  • 不能指定压缩后文件的名字，默认文件名加.gz

2，gunzip 01.py.gz

  • 不能指定名字和解压后的路径

3，配合打包解包使用

  • tar -zcvf会调用gzip，先打包，后压缩打包后的文件
  • tar -zxvf也一样

4，bzip2，调用tar -jcvf可以调用bzip2压缩解压程序而不是gzip

软件安装

1，安装，sudo apt install 软件名

2，删除，sudo apt remove 软件名

3，设置源，源就是保存了apt所有安装包的仓库

  • 设置为阿里云镜像 sudo sed -i ‘s/archive.ubuntu.com/mirrors.aliyun.com/g’ /etc/apt/sources.list
    • sed表示直接对文件做流式操作 s代表替换，把文件中archive.ubuntu.com替换为mirrors.aliyun.com，g代表全局
  • 设置 安全镜像阿里云 sudo sed -i ‘s/security.ubuntu.com/mirrors.aliyun.com/g’ /etc/apt/sources.list
    • 安全镜像负责安装后软件的安全和漏洞更新
  • 重新设置apt源的索引 sudo apt update，更新apt索引中软件的信息
    • sed：文本编辑工具
    • -i：直接修改原文件
    • 's/archive.ubuntu.com/mirrors.aliyun.com/g'：替换规则
    • /etc/apt/sources.list：要修改的文件
  • 软件是用户共用的，需要root权限才能操作

4，更新软件 sudo apt upgrade，真正更新软件版本
